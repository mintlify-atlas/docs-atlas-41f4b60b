---
title: TippySingleton component
description: Component for creating a singleton instance that controls multiple tooltips
---

The `<TippySingleton>` component creates a single tooltip that smoothly transitions between multiple trigger elements. This is useful when you have multiple tooltips in close proximity and want to provide a smoother user experience.

## Overview

A singleton allows one tooltip to be shared across multiple reference elements. When you hover over different elements, the same tooltip moves between them with a smooth transition, rather than creating and destroying separate tooltip instances.

## Basic usage

```vue
<template>
  <TippySingleton placement="top" move-transition="transform 0.2s ease-out">
    <Tippy content="First tooltip">
      <button>Button 1</button>
    </Tippy>
    
    <Tippy content="Second tooltip">
      <button>Button 2</button>
    </Tippy>
    
    <Tippy content="Third tooltip">
      <button>Button 3</button>
    </Tippy>
  </TippySingleton>
</template>
```

<Note>
  All `<Tippy>` components must be direct children of `<TippySingleton>` and must have the `data-v-tippy` attribute (automatically added by the Tippy component).
</Note>

## Props

The `<TippySingleton>` component accepts all standard [Tippy.js props](https://atomiks.github.io/tippyjs/v6/all-props/). These props control the shared behavior of all tooltips in the singleton group.

### Common singleton props

<ParamField path="moveTransition" type="string" default="undefined">
  CSS transition to apply when the tooltip moves between different reference elements.
  
  ```vue
  <TippySingleton move-transition="transform 0.2s ease-out">
    <!-- tooltips -->
  </TippySingleton>
  ```
</ParamField>

<ParamField path="placement" type="string" default="top">
  Preferred placement for all tooltips in the group. Options: `top`, `bottom`, `left`, `right`, `auto`, and variations with `-start` and `-end` suffixes.
</ParamField>

<ParamField path="arrow" type="boolean | string | Element" default="true">
  Whether to show an arrow for the singleton tooltip.
</ParamField>

<ParamField path="animation" type="string | false" default="fade">
  The animation to use for the singleton. Can be `fade`, `shift-away`, `shift-toward`, `scale`, `perspective`, or `false` to disable.
</ParamField>

<ParamField path="theme" type="string" default="''">
  Custom theme name for styling the singleton tooltip.
</ParamField>

<ParamField path="delay" type="number | [number, number]" default="0">
  Delay in milliseconds before showing/hiding. Can be a single number or `[show, hide]` tuple.
</ParamField>

<ParamField path="duration" type="number | [number, number]" default="[300, 250]">
  Duration of the animation in milliseconds. Can be a single number or `[show, hide]` tuple.
</ParamField>

<ParamField path="maxWidth" type="number | string" default="350">
  Maximum width of the tooltip in pixels or CSS unit.
</ParamField>

<ParamField path="offset" type="[number, number]" default="[0, 10]">
  Offset of the tooltip from the reference element as `[skidding, distance]`.
</ParamField>

<ParamField path="allowHTML" type="boolean" default="false">
  Whether to allow HTML content in the tooltips.
</ParamField>

<ParamField path="interactive" type="boolean" default="false">
  Makes the tooltip interactive, allowing mouse events inside it without hiding.
</ParamField>

<ParamField path="interactiveBorder" type="number" default="2">
  Size of the invisible border around the interactive tooltip.
</ParamField>

<ParamField path="interactiveDebounce" type="number" default="0">
  Debounce delay in milliseconds for interactive tooltips.
</ParamField>

<ParamField path="appendTo" type="Element | Function" default="document.body">
  The element to append the tooltip to.
</ParamField>

<ParamField path="zIndex" type="number" default="9999">
  The z-index of the tooltip.
</ParamField>

<ParamField path="trigger" type="string" default="mouseenter focus">
  Event(s) that trigger the tooltip. Options: `mouseenter`, `focus`, `click`, `manual`, or space-separated combinations.
</ParamField>

## How it works

The `<TippySingleton>` component:

1. Renders its child components (typically `<Tippy>` components)
2. On mount, finds all elements with the `data-v-tippy` attribute within its parent
3. Collects all Tippy instances from those elements
4. Creates a singleton instance that manages those tooltips
5. The singleton shows a single tooltip that transitions between triggers

## Template refs

You can access the singleton instance using template refs:

```vue
<template>
  <TippySingleton ref="singletonRef">
    <Tippy content="Tooltip 1">
      <button>Button 1</button>
    </Tippy>
    <Tippy content="Tooltip 2">
      <button>Button 2</button>
    </Tippy>
  </TippySingleton>
</template>

<script setup>
import { ref } from 'vue'

const singletonRef = ref()

// Access singleton methods
const updateSingleton = () => {
  singletonRef.value?.singleton?.setProps({
    placement: 'bottom'
  })
}
</script>
```

Exposed properties:
- `singleton` - The Tippy.js singleton instance
- `instances` - Array of child Tippy instances

### Dynamic content

```vue
<template>
  <TippySingleton 
    placement="top"
    move-transition="transform 0.2s ease-out"
  >
    <div class="items">
      <Tippy 
        v-for="item in items" 
        :key="item.id"
        :content="item.description"
      >
        <button>{{ item.name }}</button>
      </Tippy>
    </div>
  </TippySingleton>

<script setup>
import { ref } from 'vue'

const items = ref([
  { id: 1, name: 'Item 1', description: 'Description for item 1' },
  { id: 2, name: 'Item 2', description: 'Description for item 2' },
  { id: 3, name: 'Item 3', description: 'Description for item 3' },
])
</script>
```

## Best practices

- Use singletons when you have multiple tooltips in close proximity (like navigation menus or toolbars)
- Set a `moveTransition` to create smooth movement between triggers
- Keep tooltip content concise since the same tooltip is reused
- Ensure all child `<Tippy>` components have content defined
- Avoid mixing interactive and non-interactive tooltips in the same singleton

## Differences from regular Tippy

| Feature | Regular Tippy | TippySingleton |
| --- | --- | --- |
| Instances | Multiple tooltips | Single shared tooltip |
| Animation | Each tooltip animates in/out | Transitions between triggers |
| Content | Can be complex/interactive | Best with simple content |
| Use case | Individual tooltips | Groups of related tooltips |
| Performance | More DOM elements | Single tooltip, better performance |
